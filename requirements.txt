**`src/tools.py`:**
- All your `@tool` decorated functions
- Keep each tool focused and well-documented
- Each tool should handle its own error cases

**`src/prompts.py`:**
- System prompts
- Any prompt templates
- Keeps your agent.py clean

**`src/config.py`:**
- Model names
- Default settings (like temperature, max_tokens)
- File paths
- Anything you might want to change without digging through code

**`src/agent.py`:**
- Agent initialization
- Tool registration
- Main logic for running the agent
- Keep it clean â€” just orchestration, not implementation

**`main.py`:**
- Entry point with the interactive loop
- Argument parsing (if you add CLI options later)
- High-level user interaction

**`data/`:**
- Your JSON files for persistence
- Gets created on first run if it doesn't exist
- `.gitignore` the actual data files, but you could commit empty templates
